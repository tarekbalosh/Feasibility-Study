# وثيقة خطة التواصل والتقارير (Communication Plan Document)
## منصة SaaS الذكية لإنشاء دراسات الجدوى بالذكاء الاصطناعي (Feasibility Suite)

---

تهدف هذه الوثيقة إلى تنظيم التواصل وتدفق المعلومات بين أعضاء الفريق الفني والشركاء المعنيين بالمشروع (Stakeholders)، لضمان معالجة العقبات بسرعة، وتنسيق الجهود طوال فترة التطوير البالغة **14 أسبوعاً**.

---

### 1. قنوات الاتصال المعتمدة (Communication Channels)

| القناة (Channel) | الغرض والاستخدام | المستخدمون |
| :--- | :--- | :--- |
| **Slack / Discord** | - النقاشات اليومية السريعة والطارئة.<br>- التنبيهات التلقائية من GitHub للـ Commits و Pull Requests. | الفريق البرمجي بالكامل ومدير المشروع |
| **Zoom / Google Meet** | - الاجتماعات الدورية (التنسيقية والعروض التجريبية).<br>- اجتماعات التخطيط والمراجعة في نهاية كل مرحلة. | كامل الفريق بالإضافة إلى المستثمر/العميل |
| **GitHub Issues / Projects** | - إدارة المهام اليومية (Task Tracking).<br>- متابعة الأخطاء المكتشفة والإصلاحات والتطوير الفني. | الفريق البرمجي بالكامل ومدير المشروع |
| **البريد الإلكتروني (Email)** | - إرسال التقارير الرسمية الأسبوعية.<br>- مناقشة الأمور التعاقدية والمالية مع الشركاء. | مدير المشروع، مدير المنتج والشركاء |

### 2. مصفوفة الاجتماعات الدورية (Meeting Schedule)

لتفادي هدر الوقت والحفاظ على الإنتاجية، تم اعتماد الاجتماعات المركزة التالية:

#### 1. الاجتماع التنسيقي الأسبوعي (Weekly Sync Meeting)
* **الموعد:** يوم الأحد من كل أسبوع، الساعة 10:00 صباحاً (بتوقيت الرياض).
* **المدة:** 30 دقيقة كحد أقصى.
* **الأجندة:** استعراض ما تم إنجازه الأسبوع الماضي، والمهام المقررة للأسبوع الحالي، وتحديد العقبات (Blockers) لحلها فوراً.

#### 2. عرض التقدم والعرض التجريبي (Bi-weekly Demo & Review)
* **الموعد:** يوم الخميس مرة كل أسبوعين (نهاية الأسابيع الزوجية: 2، 4، 6، 8، 10، 12، 14) الساعة 3:00 مساءً.
* **المدة:** 45 - 60 دقيقة.
* **الأجندة:** استعراض الميزات التي تم تطويرها بشكل حي وتجريبي (Live Demo) أمام العميل/الشركاء والحصول على التغذية الراجعة (Feedback).

#### 3. اجتماع التخطيط للمرحلة القادمة (Phase Planning)
* **الموعد:** نهاية كل مرحلة زمنية (حسب الخطة الزمنية للمشروع).
* **المدة:** 45 دقيقة.
* **الأجندة:** فتح نطاق عمل المرحلة التالية وتوزيع المهام وتأكيد جاهزية المتطلبات.

### 4. الأدوار والمسؤوليات في التواصل (Roles & Responsibilities)

* **مدير المشروع التقني (Technical Project Manager):** المسؤول عن إرسال التقارير الأسبوعية، تنظيم الاجتماعات، حل النزاعات وتحديث الجدول الزمني وتتبع التبعيات.
* **المهندس المعماري/مطور الخلفية (Backend Lead):** المسؤول عن إبلاغ الفريق بتحديثات الـ API، تغييرات هيكل قاعدة البيانات، والتنسيق في حال توقف سيرفرات التطوير.
* **مطور الواجهة الأمامية (Frontend Lead):** المسؤول عن تقديم عروض الـ Demo وتأكيد مطابقة الكود لواجهات Figma المعتمدة.
* **مهندس فحص الجودة (QA Engineer):** المسؤول عن تسجيل الأخطاء (Bugs) على GitHub وتحديث حالة تذاكر الفحص والتنسيق مع المطورين لحلها.
* **العميل / المستثمر (Client/Product Owner):** المسؤول عن تقديم الملاحظات في اجتماعات الـ Demo والموافقة على التغييرات في نطاق العمل.
